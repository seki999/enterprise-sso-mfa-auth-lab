# 02_詳細設計書

## 画面一覧

`/`, `/login`, `/mfa`, `/dashboard`, `/admin/users`, `/admin/applications`, `/admin/policies`, `/admin/logs`, `/vpn-login`, `/docs-view`

## API一覧

| API | 用途 |
|---|---|
| GET /health | 稼働確認 |
| POST /auth/login | Mock SSO |
| POST /auth/mfa/verify | Mock MFA |
| POST /auth/logout | ログアウト |
| GET /me | ログインユーザー |
| GET /users | ユーザー一覧 |
| GET /users/{user_id} | ユーザー詳細 |
| GET /applications | アプリ一覧 |
| GET /policies | ポリシー一覧 |
| GET /auth-logs | 認証ログ |
| POST /vpn/login | VPN認証想定 |
| GET /dashboard/summary | ダッシュボード |

## DBテーブル設計

users、groups、applications、policies、auth_logs、sessionsをSQLAlchemyで定義します。

## 認証状態管理

ログイン成功時に `mfa_pending` セッションを作成し、MFA成功後に `authenticated` へ更新します。

## Mock SSO設計

ユーザーID、パスワード、アカウント状態を検証し、失敗時は認証ログに記録します。

## Mock MFA設計

6桁コード `123456` または承認ボタンで成功とします。

## Duo OIDC連携設計

`.env` のOIDC設定を利用する拡張口を用意します。未設定時はMockモードで必ず動作します。

## ポリシー判定ロジック

グループ、アプリケーション、アクセス条件をもとにMFA要否とアクセス可否を判断する想定です。

## ログ出力仕様

成功、失敗、拒否、ロックをSQLiteへ保存し、管理画面で検索します。
